Level Two

Things You Need to Know - File Extensions & Base64 Encoding

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.

File extensions tell the computer what to do with a file. For example, a HTML file will typically have a .html extension. (To enable showing file extensions in a Windows OS refer to the following link: Show or hide file name extensions). However, extensions can be arbitrarily applied. One of the reasons why someone would want to change a file's extension is to mask its true purpose. 

A file extension is not an accurate way to identify the contents of a file. So how should we identify them? By using File Signatures. But that is a topic for another day.

A simple way to look at the raw contents of a file is by opening it in a text editor instead of its default application.


Encoding in computing is to convert information in one form to another. Decoding is converting that information back into its original form. Base64 Encoding is typically used to convert binary data into text format.

Information encoded into Base64 can have the following characters: A-Z, a-z, 0-9, +, and /. The '=' character can be optionally used as padding.

So Base64 encoded output would only have those characters listed and may have the '=' character at the end. These characteristics should help you identify Base64 encoded information and prompt you to decode it to obtain the original.

Level Four

Things You Need to Know - HTTP Cookies and Ciphers

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.


HTTP is the underlying protocol used by the World Wide Web. When we enter a URL in our browser, it makes a HTTP request to the web application server and the server provides a HTTP respond.

HTTP is fundamentally a stateless protocol. This means that by itself, if a user were to login to a web application (by sending a HTTP request), and then refresh the page (send another HTTP request), the server would ask the user to login again.

This issue creates the need for a way to maintain sessions for individual users. This is where cookies come in. Servers will return HTTP responses asking the user's browser to store a cookie. This cookie is then sent along with every HTTP request the browser makes to the server. This allows the server to identify individual users based on their cookies.

Ciphers are used to conceal messages for transportation. Classical ciphers were invented before computers, and were often used by armies to transport secret messages.

Classical ciphers are mostly no longer in use as they aren't very secure for protecting information (with today's computers to break them). However, it is still interesting to know how people used to protect information. Here are a list of classical ciphers.

Level Five

Things You Need to Know - Binary Number & Image Steganography

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.
Binary numbers are strings of 0's and 1's. Inside a computer, all data is stored in binary. This is because of the way electronics represent states, 0's represent 'off' (no electrical current) and 1's represent 'on' (has electrical current).

For example, the letter 'A' that we see on screen is stored as 01000001 in binary.

Steganography is used to hide information to prevent detection of a hidden message. Image Steganography involves hiding information in an image. Effective image Steganography should result in an image which is visually similar to the original image.

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.
When we communicate through the Internet, either by sending messages or files, the data to be sent is broken down into chunks and sent independently. It is then reassembled at the destination. These chunks are packets

 It is possible to capture these packets on a network interface on your machine. Wireshark is a tool that is capable of doing this. It is also the go-to tool for analyzing captured packets.

 One feature I find useful when analyzing captured packets is the conversations window which can be found under 'Statistics -> Coversations'. This window will show you a summary of all the network conversations.

You can then select a conversation and click on 'follow stream'. This will show you the full conversation between 2 points instead of having to go through each packet one by one.

 ASCII is used to represent alphanumeric characters as numbers. This is necessary as computers operate in binary. 

 For example, the letter 'N' in:

Level Nine

Things You Need to Know - Authentication Risks

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.

There are inherent risks with authentication systems in web applications. Here are some of the most commonly found risks.

SQL Injection is where user input is unsafely incorporated into SQL queries. This applies to web applications which make use of SQL queries to obtain data from a database. Poorly designed code can lead to SQL injection where attackers can easily bypass the authentication system even without a password. 

Use of weak passwords is something very common. This makes it easy for an attacker to guess a user's password. For example, these are the top 5 more commonly used passwords in 2014
1. 123456 (Unchanged)
2. password (Unchanged)
3. 12345 (Up 17)
4. 12345678 (Down 1)
5. qwerty (Down 1)
Another risk is the use of default passwords, networking equipments such as routers often come with a web interface. Manufacturers often provide a set of default credentials so that users can access the web interface during first time use. 

 However, many of these credentials are never changed. This makes it a huge risk as the default credentials can be easily obtained. Attackers can just try using the default credentials to access these systems.

Level Thirteen

Things You Need to Know - Backup Files & File Signatures (aka Magic Numbers)

This section should provide you with the basic concepts needed to solve this level without looking at the solution below.

There are times when developers create backup files in the same directory and forgot to remove them. This provides a source of information for attackers if there are any sensitive information left behind in the backup files. 

Access to these files are often not restricted and can be directly accessed by requesting the file directly. For example, instead of requesting for 'login.html', attempt to request for 'login.html.backup'.

Files in computers can be identified by their file signatures (aka Magic Numbers). Special numbers are stored at the beginning (and sometimes end) of a file. These numbers are used as markers to identify the type of file.

A list of file signatures can be found at: http://www.garykessler.net/library/file_sigs.html

If you have a Linux/Unix system, you can use the 'file' command to determine file types based on the system's 'file signature' file.

 Shell Injection happens because user input is not properly validate before being used in a command executed by the OS. This is a critical vulnerability as it can allow attackers to execute arbitrary commands.

There are many ways people encrypt/encode passwords used to download files. These are a few types of the encrypting/encoding algorithms:

U-ASCII Encode, T-ASCII Encode, F-ASCII Encode, AER-256 Encrypt, ARMON-64 Encrypt, ATOM-128 Encrypt, BASE-64 Encrypt, ESAB-46 Encrypt, EZIP-64 Encrypt, FERON-74 Encrypt, GILA7 Encrypt, HAZZ-15 Encrypt, MEGAN-35 Encrypt, OKTO3 Encrypt, TIGO-3FX Encrypt, TRIPO-5 Encrypt, ZARA-128 Encrypt, ZONG22 Encrypt

Some of these have the same output characters as Base64 and may look like Base64, but when decoded using Base64, don't provide meaningful results.

This website http://www.crypo.com/ can encrypt and decrypt many of the algorithms listed above.
